# Ren'Ts

A generic Ren'Py visual-novel story reader built with **Bun + React**.  
Script files are stored in the project's own `.rrs` (Ren'Ts Script in Rust style) format, converted from original `.rpy` sources by the included toolchain.

## .rrs Script Format

Scripts are stored as `.rrs` files in `assets/data/`, loaded at startup via `assets/data/manifest.json`.

Each `.rrs` file begins with optional top-level declarations followed by one or more `label` blocks. Global declarations are written as plain bare assignments (no keyword) — outside a `label` block they become global defines; inside a `label` block they become save-file variables:

```
// Character name definitions (auto-generated by rpy2rrs)
char.k  = "Keitaro";
char.hi = "Hiro";

label day1 {
  scene "BGs/campsite_day.jpg" | dissolve;
  music::play("Audio/BGM/Outdoors.ogg");

  show keitaro_casual::normal1 @ center;
  speak k "We made it!";

  menu {
    "Go find Hiro." => {
      jump day1_hiro;
    }
    "Look around." => {
      jump day1_explore;
    }
  }
}
```

**Full syntax reference, statement guide, and examples: [tools/rrs/README.md](tools/rrs/README.md)**

## Ren'Py → .rrs Converter

### Command-line tool (recommended for batch conversion)

```bash
# Batch-convert an entire game directory, generate manifest.json
bun run rpy2rrs /path/to/game/ -o assets/data/ --manifest

# With Chinese translation merging
bun run rpy2rrs /path/to/game/ -o assets/data/ --manifest \
  --tl /path/to/game/tl/chinese/

# Specify a game name written into manifest.json
bun run rpy2rrs /path/to/game/ -o assets/data/ --manifest \
  --game "My VN Game"
```

The converter automatically:
- Detects `Character("Name")` definitions and emits `define char.*` headers
- Parses audio / background / CG / SFX asset maps from `script.rpy`
- Converts `label start` (and all other labels) from `script.rpy`
- Generates a `manifest.json` with `start`, `game`, and `files` fields

**Full options and workflow: [tools/rrs/README.md](tools/rrs/README.md)**

### In-browser converter

A conversion UI is built into the app.  Click the **🔄 转换 .rpy 文件** button on the title screen to open it.  You can drag-and-drop `.rpy` files (including `script.rpy` for full asset-map resolution), configure options, and download the converted `.rrs` files directly from the browser.

## manifest.json Format

```json
{
  "start": "start",
  "game": "My VN Game",
  "files": ["script.rrs", "day1.rrs", "day2.rrs"]
}
```

- `start` — entry label name; defaults to `"start"` (Ren'Py convention)
- `game` — optional game title displayed on the title screen
- `files` — all `.rrs` files containing at least one label

## Engine Architecture

```
assets/data/*.rrs
       │
       │  loader.ts: reads manifest.json, loads all .rrs files in parallel
       │  rrs/index.ts: parseScript() → ScriptFile { labels: Record<string, Step[]> }
       ▼
  engine.ts ── pure-function state machine
  · advance(state, action) → GameState
  · runUntilBlocked(): executes instant steps until a blocking step is reached
  · Blocking steps (wait for user input): speak / menu / pause
  · Instant steps (auto-execute): scene / show / hide / expr / music / sound /
                                   assign / if / jump / call / return
       │
       ▼
  store.ts ── Zustand state + side-effect dispatch
  · calls engine.ts pure functions to advance state
  · drives audio.ts for audio management
  · calls save.ts for auto-save
       │
       ▼
  React component tree (reads store, contains no logic)
  · Background.tsx   ── background layer (image / colour / video)
  · SpriteLayer.tsx  ── sprite layer (characters + CGs)
  · DialogueBox.tsx  ── dialogue box (typewriter effect)
  · ChoiceMenu.tsx   ── choice menu
  · Toolbar.tsx      ── toolbar (save / gallery / settings)
  · ConvertScreen.tsx── .rpy → .rrs conversion overlay
```

## Save System

| Mechanism | Description |
|-----------|-------------|
| Auto-save | Written to `localStorage` on every dialogue step |
| File export | Current state serialised to `.json`, triggers browser download |
| File import | Reads a `.json` save file and restores game progress |
| Tauri auto-save | In desktop mode, auto-saves to a user-chosen native file |
